# Day 0: Why I'm Doing This

**Date:** 2026-02-06
**TL;DR:** I'm committing to being AI-first as a developer and documenting the entire journey from the start — the good, the bad, and the embarrassing.

---

## The Problem

My company has a Slack channel called `dev-ai-guild`. It's full of engineers who are clearly ahead of the curve — talking about custom MCP servers, agent workflows, prompt engineering techniques, new models, and how they're 5x more productive.

And every time I read those messages, I feel a little more behind.

I know I'm not alone. I've talked to other engineers who feel the same way. They use AI — maybe ChatGPT for quick questions, maybe Copilot for autocomplete — but they know they're barely scratching the surface. The problem is, when the only voices in the room are the experts, it's hard to ask "basic" questions without feeling stupid.

So the experts keep talking to each other, and everyone else stays quiet.

## What I'm Going to Do About It

I'm going to learn in public. Starting from wherever I am right now — which is not very far — and documenting everything:

- What I tried
- What worked
- What didn't
- What confused me
- What finally clicked

No polishing, no pretending I knew things before I did. Just honest notes from a developer figuring this out in real time.

## Where I'm Starting From

I'm a software engineer. My day-to-day stack is TypeScript, React Native, React, and Ruby on Rails. I've been using AI tools casually:

- ChatGPT for questions and brainstorming
- Copilot for autocomplete
- Claude Code (recently) for coding tasks in the terminal

But I wouldn't call myself proficient with any of them. I use them the way I used Stack Overflow in 2015 — as a crutch, not a tool I actually understand.

Today I'm changing that. I'm committing to being **AI-first**, meaning: before I do something the old way, I'll try the AI way first. If it's faster, great. If it's not, I'll document why.

## Why Public?

Because I think there are a lot of developers in my position who would benefit from seeing someone else go through this. Not a tutorial from an expert. Not a Twitter thread from someone who's been doing this for two years. Just a regular developer, starting from scratch, sharing what they learn each day.

If that's you, follow along. Ask questions. Share your own experiences. Let's figure this out together.

## What's Next

Tomorrow I'm going to start using Claude Code as my primary coding tool and see how it changes (or doesn't change) my workflow. I'll report back.
